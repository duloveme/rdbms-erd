# RDBMS ERD Designer - 개발 순서 및 진행상황

## 목적

- `docs/DEVELOPMENT_PLAN.md`의 스펙을 실제 구현 순서로 풀어낸 실행 문서.
- 의존관계를 명확히 해서 "무엇을 먼저 만들어야 다음 단계가 가능한지"를 관리.

## 운영 규칙

- 각 단계는 **완료 조건(Definition of Done)** 을 만족하면 완료로 처리.
- 단계 내 세부 항목은 이슈 트래커로 분할하고, 이 문서에는 핵심 게이트만 유지.
- 스코프 변경 시 먼저 본 문서를 갱신하고, 이후 구현을 진행.
- 진행상태는 **아래 단계별 체크박스만** 업데이트한다(별도 체크리스트/상태표 미사용).
